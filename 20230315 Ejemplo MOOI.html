Ejemplo de la metodología MOOI¶

El MOOI es un proceso creado por el Observatorio de Desarrollo Económico de Bogotá (ODEB) para optimizar las convocatorias de la oferta institucional de la Secretaría Distrital de Desarrollo Económico (SDDE) y facilitar su evaluación. Para esto, el MOOI utiliza una base única de unidades productivas de Bogotá y estima un puntaje que organiza las unidades productivas de acuerdo con la posibilidad que estas tienen de beneficiarse del programa. Primero, se filtra la base de datos para dejar solamente las unidades productivas elegibles para el programa. Luego, se realiza un análisis de factores para datos mixtos, una variante de la metodología de componentes principales, para reducir la dimensionalidad de los criterios de priorización de las unidades productivas participantes. Posteriormente, agrupa las unidades productivas de acuerdo a esta caracterización simplificada usando la técnica k-means y, por último, genera un puntaje de priorización de acuerdo a la distancia de la unidad productiva al centroide del grupo preferido para recibir el programa.

El programa con el que se realizará el análisis es Camino a la inclusión financiera, el cual es parte de la estrategia de la SDDE con Bancoldex para que las unidades productivas accedan a créditos. En concreto, es un curso de formación en finanzas para los micronegocios el cual, una vez completado, le da a los participantes un certificado con el que pueden ir a las entidades financieras y acceder a los beneficios de Bancoldex.

Nota aclaratoria: El ejemplo aquí presentado está hecho con fines indicativos y no para ser puesto en operación. Especialmente porque, si bien lso criterios de inclusión son acertados, los criterios de priorización aún no han sido validados con la dirección implementadora.

Selección de variables de exclusión y priorización¶

Se toman los criterios de los programas, se filtra la base y se dejan las variables de priorización en la forma deseada

Existen dos tipos de criterios a tener en cuenta: a. Los de exclusión: las características que necesita tener una unidad productiva para acceder al programa b. Los de priorización: aquellas características que, de acuerdo a la teoría de cambio, hacen que una unidad productiva elegible saque mayor provecho del programa.

Para Camino a la inclusión financiera se tienen: a. Criterios de exclusión: ser un micronegocio (tener un máximo de 9 personas trabajando). b. Criterios de priorización: no llevar una contabilidad (porque pueden sacar mayor provecho de una formación en finanzas), llevar más tiempo funcionando (son negocios más sólidos y es más factible que puedan sobrellevar un préstamo), bajos ingresos (como proxy de productividad), expresar interés en un curso de finanzas, expresar deseos de tener un crédito y ser propiedad de mujeres (los negocios de mujeres tienen menor acceso a crédito).

Las variables de priorización aquí elegidas sirven para ejemplificar la operación del MOOI, sin embargo, hay muchas otras variables que se podrían considerar como valiosas en este análisis. Por ejemplo, el sector al que pertenece, la razón para comenzar el negocio y con qué entidades ha tenido o tiene préstamos, las cuales se encuentran en la Tropa Económica y pueden ser incluidas en el análisis, pero se dejaron por fuera para facilitar la lectura.

Función MOOI¶

A continuación se explican algunas funciones y sus usos dentro de la función del MOOI

Imputación de datos utilizando MICE¶

Algunos registros pueden tener valores faltantes, sea por negación al contestar por parte del encuestado o por errores de registros del encuestador. Dado lo anterior, y ante la incoveniencia de eliminar registros con algunas variables faltantes para no dejar por fuera a unidades productivas de los programas, se opta por recurrir a estrategias de imputación.

Para la imputación de variables continuas se utiliza el método de Imputaciones Múltiples por Ecuaciones Encadenadas, conocido como MICE por sus siglas en inglés. Este es un método de imputación iterativo el cual imputa valores faltantes para cada columna utilizando una regresión bayesiana lineal y como variables independientes todas las demás columnas. Este proceso lo realiza manera iterativa hasta minimizar la diferencia en valores de una iteración a otra.

A continuación se define los criterios del modelo. 1. Como ecuación para estimación se utiliza una regresión bayesiana lineal 2. En la primera iteración se utiliza la media de las variables para la regresión 3. Se imputa primero tomando las variables con menor cantidad de valores faltantes como variable dependiente.

Reducción de dimensiones utilizando FAMD¶

Para reducir la dimensionalidad tanto de variables continuas como categóricas se utiliza el análisis de factores para datos mixtos (FAMD en inglés). Al igual que el PCA, este análisis parte del principio de que existen unos componentes que condensan la varianza de múltiples variables. A diferencia del PCA, el FAMD tiene en cuenta la varianza tanto de variables continuas como categóricas.

# FAMD: reducción de dimensiones

            n_iter=10,
            copy=True, 
            check_input=True,
            engine='auto', 
            random_state=42)

Clusterización utilizando K-means¶

K-means es una de las metodologías más usadas para hallar clústeres o cúmulos de datos y su uso es tan extendido porque esos grupos tienen la característica de que sus miembros son lo más parecido posible entre sí. El algoritmo comienza con una agrupación aleatoria de las observaciones en un número de agrupaciones preestablecido y minimiza las distancias de cada observación a su centroide, mientras maximiza la distancia entre los centroides. El algoritmo recursivamente reagrupa las observaciones hasta que encuentra los centroides y grupos óptimos (centroides), así, una observación cualquiera pertenece a un grupo porque está más cerca del centroide que lo representa.

Ejecución del proceso¶

Entonces, en el MOOI se realizan los siguientes pasos:

a) Filtros de inclusión: selección de las variables según los criterios de participación del programa. b) Caracterización de unidades productivas: reducción de dimensiones de la base resultante (FAMD): este NO ES el puntaje final c) Agrupación de unidades productivas según características: clusterización de las dimensiones (k-means y elbow) d) Elección del perfil más adecuado entre los grupos: ojo de experto, de acuerdo a variables más importantes e) Asignación de puntaje a cada potencial beneficiario: distancia al centroide: este ES el puntaje final
